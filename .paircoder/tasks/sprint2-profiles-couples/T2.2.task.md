## Objective
Implement 4 REST endpoints for couple CRUD on the Cloudflare Worker against D1.

## Acceptance Criteria
- [x] POST /api/couples creates couple, returns 201
- [x] POST validates both partner IDs exist, returns 400/404 if not
- [x] POST with already-linked partner returns 409
- [x] GET /api/couples lists all couples
- [x] GET /api/couples/:id returns couple with nested partner profiles (JOIN)
- [x] GET /api/couples/:id returns 404 for unknown id
- [x] DELETE /api/couples/:id removes couple, preserves profiles
- [x] DELETE returns 404 for unknown id
- [x] 9 tests pass

## Implementation Details
- File: `worker/src/routes/couples.ts`
- Mount: `api.route("/couples", coupleRoutes)` in index.ts
- GET :id uses JOIN to return denormalized couple+partners response
- UNIQUE constraint on partner_a/partner_b catches double-linking
