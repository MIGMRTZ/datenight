## Objective
Implement 5 REST endpoints for partner profile CRUD on the Cloudflare Worker against D1.

## Acceptance Criteria
- [x] POST /api/profiles creates profile, returns 201
- [x] POST validates required fields (name, cuisines, movie_genres, activities), returns 400
- [x] POST with duplicate id returns 409
- [x] GET /api/profiles lists all profiles with parsed JSON arrays
- [x] GET /api/profiles/:id returns single profile or 404
- [x] PUT /api/profiles/:id updates profile and bumps updated_at, or 404
- [x] DELETE /api/profiles/:id removes profile or 404
- [x] DELETE returns 409 if partner is in a couple
- [x] Timestamps (created_at, updated_at) set server-side
- [x] 12 tests pass

## Implementation Details
- File: `worker/src/routes/profiles.ts`
- Mount: `api.route("/profiles", profileRoutes)` in index.ts
- JSON arrays stored as TEXT, parsed on read with JSON.parse()
- DELETE checks couples table before removing
