## Objective
Build pydantic-settings based config that reads `config.yaml` + environment variable overrides. Auth token is env-only.

## Acceptance Criteria
- [x] `datenight/config.py` has nested Pydantic models for all config sections
- [x] `load_settings()` reads config.yaml, falls back to defaults
- [x] `DATENIGHT_AUTH_TOKEN` env var populates `auth_token`
- [x] `DATENIGHT_LOCATION__ZIP` style nested env overrides work
- [x] Missing config.yaml gracefully uses defaults
- [x] 7 tests pass

## Implementation Details

**Nested models:** LocationConfig, OllamaConfig, CloudflareConfig, CalendarConfig, PlanningConfig, LoggingConfig

**Config loading:**
1. Look for `config.yaml` in CWD or `~/.datenight/config.yaml`
2. Parse with PyYAML
3. Pass dict to `DateNightSettings(**yaml_data)`
4. Env vars auto-override via pydantic-settings (`DATENIGHT_` prefix, `__` nesting)
